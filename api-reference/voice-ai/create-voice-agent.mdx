# Create Voice Agent API

This documentation explains how to create a new voice agent using the API.

## Endpoint

**POST** `/api/v1/global/agents`

## Authentication

Include your API token in the request header:

```
token: <your_api_token>
```

or

```
api_access_token: <your_api_token>
```

## Request Body

```json
{
  "agent_name": "Customer Support Agent",
  "description": "Handles customer inquiries",
  "agent_prompt": "You are a helpful customer support agent. Answer questions politely and professionally.",
  "voice_id": "RXe6OFmxoC0nlSWpuCDy",
  "voice_provider": 1,
  "transcriber_provider": 2,
  "transcriber_language": 1,
  "transcriber_model": 16,
  "calling_type": 1,
  "purpose": 2
}
```

## Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `agent_name` | string | Name of the voice agent |
| `agent_prompt` | string | System prompt/instructions for the agent. This defines how the agent behaves and responds |

## Optional Fields

| Field | Type | Description | Default |
|-------|------|-------------|---------|
| `description` | string | Description of the agent | Empty string |
| `voice_id` | string | Voice ID to use for the agent. Use the voice ID from the voices API | "RXe6OFmxoC0nlSWpuCDy" |
| `voice_provider` | integer | Voice provider ID (see Voice Providers below) | 1 (ElevenLabs) |
| `transcriber_provider` | integer | Speech-to-text provider ID (see Transcriber Providers below) | 2 (Deepgram) |
| `transcriber_language` | integer | Language ID for transcription (see Languages below) | 1 (English) |
| `transcriber_model` | integer | Transcriber model ID (see Transcriber Models below) | 16 (nova-3) |
| `calling_type` | integer | Calling type (see Calling Types below) | 1 (Outbound) |
| `purpose` | integer | Purpose ID for the agent (see Purpose Options below) | None |
| `tts_model` | string | TTS (Text-to-Speech) model for voice synthesis | "eleven_multilingual_v2" |

## Calling Types

**Important:** The `calling_type` field determines whether the agent makes outbound calls or receives inbound calls.

| ID | Type | Description |
|----|------|-------------|
| 1 | **Outbound** | Agent makes calls to customers. Use this when you want the agent to initiate calls to phone numbers. |
| 2 | **Inbound** | Agent receives calls from customers. Use this when customers call your phone numbers and the agent answers. |

**Note:** 
- For outbound calling (making calls to customers), use `calling_type: 1`
- For inbound calling (receiving calls from customers), use `calling_type: 2`
- Default value is `1` (Outbound) if not specified

## TTS Models

The TTS (Text-to-Speech) model determines the voice synthesis engine used for the agent.

| Model | Description |
|-------|-------------|
| `eleven_multilingual_v2` | ElevenLabs Multilingual V2 model (Default) - High-quality multilingual voice synthesis |

**Note:** The default TTS model is `eleven_multilingual_v2` when not specified. This model supports multiple languages and provides high-quality voice synthesis.

## Example Request

```bash
curl -X POST https://your-domain.com/api/v1/global/agents \
  -H "Content-Type: application/json" \
  -H "token: your_api_token_here" \
  -d '{
    "agent_name": "Support Agent",
    "description": "Customer support voice agent",
    "agent_prompt": "You are a helpful customer support agent. Be polite and professional.",
    "voice_id": "RXe6OFmxoC0nlSWpuCDy",
    "voice_provider": 1,
    "transcriber_provider": 2,
    "transcriber_language": 1,
    "transcriber_model": 16,
    "calling_type": 1,
    "purpose": 2
  }'
```

## Notes

- The `voice_id` is optional. If not provided, it defaults to "RXe6OFmxoC0nlSWpuCDy" (Anika - Soothing Customer Care Agent). You can fetch available voices using the voices API endpoint.
- The `voice_id_json` is automatically set with the default voice information when `voice_id` is not provided. The default JSON contains the complete voice metadata including ID, title, description, URL, and provider information.
- The `tts_model` is optional. If not provided, it defaults to "eleven_multilingual_v2".
- **Important:** The agent will be created with status `3` (Draft) by default. To activate the agent and make it available for calls, you must call the Update Agent API with `status: 1` after creation. See the Update Agent API documentation for details.
- The `agent_prompt` is crucial as it defines the agent's behavior and personality during conversations.
- `voice_infra` is automatically set to 3 (Internal call) and cannot be changed via the API.
- A default phone number (+918062810349) is automatically added to the agent during creation. This phone number can be used for making calls with the agent.
