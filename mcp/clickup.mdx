# ClickUp MCP

> **In one line:** Connect ClickUp so your UnleashX agent can create and update tasks, organize spaces and lists, post comments, and track work without you opening ClickUp.

| | |
|---|---|
| **Category** | Productivity |
| **Authentication** | Personal API Token |
| **Setup time** | ~5 minutes |
| **Difficulty** | Easy |
| **Best for** | Teams who want to capture, triage, and update project tasks straight from conversations |

## 1. Overview

ClickUp is a project and task management platform where work is organized into Workspaces (Teams), Spaces, Folders, Lists, and Tasks. Teams use it to plan projects, assign work, track progress, and collaborate with comments and checklists.

Once connected, your UnleashX agent can browse your ClickUp hierarchy, create and update tasks, leave comments, build checklists, and pull task details on demand. That means you can turn a phone call, chat, or instruction into a real ClickUp task and keep everything in sync automatically.

Connecting ClickUp to UnleashX is useful when you want your agent to log requests, follow up on action items, or report project status, so nothing falls through the cracks and your team stays focused on the work itself.

## 2. What you'll need

- A ClickUp account with access to the Workspace you want to use.
- Permission to view and edit the Spaces, Lists, and Tasks you want the agent to manage.
- A ClickUp **Personal API Token** (created from your own profile settings).

<Note>
You don't need to be a Workspace admin to create a Personal API Token. The token inherits **your** ClickUp permissions, so the agent can only see and change what you can. If you lack access to certain Spaces, ask an admin to grant it before connecting.
</Note>

## 3. Get your credentials

ClickUp uses a **Personal API Token** that begins with `pk_`.

<Steps>
<Step title="Open your ClickUp settings">
Sign in at [https://app.clickup.com](https://app.clickup.com), click your avatar in the lower-left corner, and choose **Settings**.
</Step>
<Step title="Go to Apps">
In the left sidebar of Settings, select **Apps**. (Direct link: [https://app.clickup.com/settings/apps](https://app.clickup.com/settings/apps))
</Step>
<Step title="Generate your token">
Under **API Token**, click **Generate** (or **Regenerate** if you already have one).
</Step>
<Step title="Copy your token">
Copy the token that begins with `pk_`.

<Warning>
Treat this token like a password. Anyone with it can act in ClickUp as you. If it leaks, return to this page and click **Regenerate** to invalidate the old one.
</Warning>
</Step>
</Steps>

**What the token can access**

| Scope / Access | Why UnleashX needs it |
|---|---|
| Read Workspaces, Spaces, Folders, Lists | To navigate your ClickUp hierarchy and find the right place to work |
| Read & write Tasks | To create, update, fetch, and delete tasks on your behalf |
| Read & write Comments | To post and read task comments |
| Read & write Checklists | To create checklists and add items to tasks |
| Read Members, Views, Goals, Templates | To resolve assignees, show views, and apply templates |

<Note>
ClickUp's Personal API Token is not scope-limited — it grants the same access you have in the app. UnleashX only calls the endpoints needed for the tools listed in Section 5.
</Note>

## 4. Connect on UnleashX

<Steps>
<Step title="Open your agent">
Go to [https://tryunleashx.com](https://tryunleashx.com) and open the agent you want to connect.
</Step>
<Step title="Open Data Connectors">
In the agent, go to **Data Connectors**.
</Step>
<Step title="Find ClickUp">
Locate **ClickUp** in the list and click **Connect** / **Add** / **Configure**.
</Step>
<Step title="Paste your token">
When prompted, paste your `pk_` **Personal API Token**. UnleashX sends it as the `clickup_api_token` / `access-token` header (Bearer).
</Step>
<Step title="Confirm success">
Save. When the status shows a **Connected** badge, your agent can start working with ClickUp.
</Step>
</Steps>

## 5. Available tools

| Tool (technical) | What it does | Changes data? |
|---|---|---|
| `clickup_get_workspaces` | Lists all Workspaces you can access | No |
| `clickup_get_team_ids` | Gets Team (Workspace) IDs | No |
| `clickup_get_spaces` | Lists Spaces in a Team | No |
| `clickup_create_space` | Creates a new Space in a Team | ✏️ Yes |
| `clickup_get_folder` | Gets a Folder by ID | No |
| `clickup_create_folder` | Creates a Folder in a Space | ✏️ Yes |
| `clickup_list_folders` | Lists Folders in a Space | No |
| `clickup_get_lists` | Lists Lists in a Space or Folder | No |
| `clickup_create_list` | Creates a List in a Folder | ✏️ Yes |
| `clickup_list_tasks` | Lists tasks in a List with paging/sorting | No |
| `clickup_create_task` | Creates a new task | ✏️ Yes |
| `clickup_get_task` | Gets a task by ID | No |
| `clickup_get_ticket` | Gets a ticket (task) by ID | No |
| `clickup_delete_task` | Deletes a task | ⚠️ Yes |
| `clickup_update_task` | Updates fields on a task | ✏️ Yes |
| `clickup_list_task_comments` | Lists comments on a task | No |
| `clickup_create_task_comment` | Adds a comment to a task | ✏️ Yes |
| `clickup_create_checklist` | Creates a checklist on a task | ✏️ Yes |
| `clickup_add_checklist_item` | Adds an item to a checklist | ✏️ Yes |
| `clickup_list_all_tasks` | Lists all tasks in a List | No |
| `clickup_list_all_spaces` | Lists all Spaces in a Team | No |
| `clickup_list_all_subtasks` | Lists all subtasks of a task | No |
| `clickup_list_all_templates` | Lists all templates in a Team | No |
| `clickup_list_task_templates` | Lists task templates in a Team | No |
| `clickup_folderless_list` | Lists folderless Lists in a Space | No |
| `clickup_list_view` | Gets views for a List | No |
| `clickup_list_members` | Lists members of a List | No |
| `clickup_get_list` | Gets a List by ID | No |
| `clickup_delete_list` | Deletes a List | ⚠️ Yes |
| `clickup_list_all_goals` | Lists all Goals in a Team | No |
| `clickup_list_space_view` | Gets views for a Space | No |
| `clickup_list_view_task` | Lists tasks within a view | No |
| `clickup_list_all_channel_members` | Lists channel members in a Team | No |
| `clickup_clear_snapshots` | Clears cached ClickUp data snapshots | ⚠️ Yes |
| `clickup_on_task_created` | Trigger that fires when a task is created in a List | No |
| `clickup_on_task_updated` | Trigger that fires when a task is updated in a List | No |

## 6. Example usage

**"Create a high-priority task in my Marketing list to follow up with the new lead by Friday."**
Behind the scenes: `clickup_get_lists` (to find the Marketing list) → `clickup_create_task`.

**"What's open in our Sprint list, and add a comment to the login bug saying I'm on it."**
Behind the scenes: `clickup_list_all_tasks` → `clickup_get_task` → `clickup_create_task_comment`.

## 7. Permissions & data access

**UnleashX CAN:**
- Read your Workspaces, Spaces, Folders, Lists, and Tasks.
- Create, update, and delete tasks, Lists, Spaces, and Folders you have access to.
- Post and read comments, and build checklists.
- Read members, views, goals, and templates.

**UnleashX CANNOT:**
- Access Workspaces or Spaces your ClickUp account can't see.
- Change ClickUp account or billing settings.
- Act beyond the permissions your own ClickUp user has.

**To disconnect:** In UnleashX, open your agent → **Data Connectors** → **ClickUp** → **Disconnect**. To fully revoke access, go to ClickUp **Settings → Apps** and click **Regenerate** to invalidate the token. Either action stops access immediately.

## 8. Troubleshooting

| Problem | What it means | How to fix it |
|---|---|---|
| `401 Unauthorized` | The API token is wrong, expired, or was regenerated | Generate a fresh token in **Settings → Apps** and reconnect in UnleashX |
| `403 Forbidden` | Your ClickUp user lacks permission for that Space/List/Task | Ask a Workspace admin to grant you access, then retry |
| `404 Not Found` | The task, list, or space ID no longer exists | Re-list items to get a current ID; the item may have been deleted |
| Rate limited | Too many requests in a short window | Wait a minute and try again; ClickUp enforces per-token rate limits |
| Stale results | A cached snapshot is out of date | Run `clickup_clear_snapshots` or ask the agent to refresh |

For general connection issues, see [/mcp/integrations](/mcp/integrations).

## 9. Frequently asked questions

**Is my data stored by UnleashX?**
UnleashX uses your token to call ClickUp live when a tool runs. It may cache short-lived snapshots to speed up navigation; you can clear them with `clickup_clear_snapshots`.

**Can multiple team members connect?**
Yes. Each person creates their own Personal API Token and connects it to their agent. Each token reflects that person's ClickUp permissions.

**How do I rotate my token?**
Go to ClickUp **Settings → Apps**, click **Regenerate**, then paste the new token into UnleashX. The old token stops working immediately.

**Can the agent delete things?**
Yes — `clickup_delete_task` and `clickup_delete_list` remove items permanently. These are marked ⚠️ in the tools table.
